What this feature does
----------------------
This feature is already wired in the app.
When a tester completes a test, the frontend calls the Supabase Edge Function:
  send-test-results-notification

That function sends the email and then marks the response as already notified.

Important
---------
The current implementation uses the SMTP2GO Email API, not raw SMTP.
So the SMTP server and port are NOT needed for this setup.
What you need instead is:
- SMTP2GO API key
- sender email address you want emails sent from

Also important:
Supabase Edge Functions already expose the built-in server credentials they need.
You do NOT need to create a custom SUPABASE_SECRET_KEY secret for this notification function.

Remaining Step 1: run the only extra SQL needed for notifications
----------------------------------------------------------------
In Supabase SQL Editor, paste and run:

alter table public.test_responses
  add column if not exists owner_notified_at timestamptz;

Or use this file:
  C:\Projects\LIKE4LIKE\supabase\test-results-email-update.txt

This statement is safe to run even if the column already exists.

Remaining Step 2: set the SMTP2GO function secrets
--------------------------------------------------
Run these from PowerShell in:
  C:\Projects\LIKE4LIKE

npx supabase secrets set SMTP2GO_API_KEY=your_smtp2go_api_key SMTP2GO_SENDER=your_sender_email APP_BASE_URL=https://test4test.io --project-ref lteimepkxuiupbcsbcpz

Example sender:
  no-reply@test4test.io

Remaining Step 3: redeploy the notification function if secrets change later
----------------------------------------------------------------------------
npx supabase functions deploy send-test-results-notification --project-ref lteimepkxuiupbcsbcpz

How to test
-----------
1. Sign in with account A and submit an app.
2. Sign in with account B and complete a test for account A's app.
3. Account A should receive an email.
4. The email should link to:
   https://test4test.io/my-tests/<submission-id>

What else you needed to provide
-------------------------------
You did NOT need to provide the SMTP server or port for the current implementation.
What was needed was:
- SMTP2GO API key
- sender email address/domain that SMTP2GO allows you to send from

Notes
-----
- The app already triggers this after successful test submission.
- Notifications are idempotent because owner_notified_at prevents duplicate sends for the same response.
- If you want, this can be rewritten later to use raw SMTP instead, but the API approach is much simpler and more reliable inside Supabase Edge Functions.
