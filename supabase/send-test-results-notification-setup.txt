Test result email notification setup for Test4Test
===============================================

This notification flow now has two parts:
1. `send-test-results-notification`
   Sends the first owner email when approved feedback arrives.
   If the owner still owes that tester a test back, this first email doubles as the stage-1 reminder.
2. `send-test-back-reminders`
   Runs on a schedule and handles the day-1 and day-2 follow-up reminders.

Use this file as the quick overview, then follow the full setup guide here:
  C:\Projects\LIKE4LIKE\supabase\test-back-reminders-setup.txt

Quick summary
-------------
- Run the latest migrations:
  - supabase/migrations/20260408_test_back_reminder_emails.sql
  - supabase/migrations/20260409_test_back_rate_grace_period.sql
- Set these function secrets:
  - SMTP2GO_API_KEY
  - SMTP2GO_SENDER
  - APP_BASE_URL
  - TEST_BACK_REMINDER_CRON_SECRET
- Deploy:
  - send-test-results-notification
  - send-test-back-reminders --no-verify-jwt
- Create the hourly pg_cron schedule from the full setup guide.

Important behavior changes
--------------------------
- The owner email now comes only from approved feedback, not flagged feedback.
- Test-back reminder copy is editable in `public.email_templates`.
- Reminder delivery history is logged in `public.email_delivery_logs`.
- The public Test-back Rate now drops when the final reminder is sent, not immediately when a new owed test-back is created.
- Reminder sequences stop automatically when the owner tests back.
